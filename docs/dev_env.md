## Установка
Для начала, чтобы установить сборку проекта, вам необходимо скачать:  

* [.NET 10.0](https://dotnet.microsoft.com/en-us/download/dotnet/10.0)  
* [Python](https://www.python.org/downloads/)
* [Git](https://git-scm.com/downloads)  
* [Visual Studio Code](https://visualstudio.microsoft.com/ru/downloads/ "Важно! Именно Code версия, что ниже обычной на странице.")  
  
  
Далее создайте [Fork](https://github.com/dead-space-server/space-erida-14/fork) от основного репозитория.  

Клонируйте его в любую директорию и запустите RUN_THIS.py в корневой папке сборки. Этот скрипт обновляет сабмодули, то есть устанавливает движок игры.

## Запуск
Когда мы только скачали сборку, её не выйдет запустить - для этого необходимо произвести билд.  

Для этого вводим в консоль `dotnet build`, после окончания которого открываем `bin/Content.Server/Content.Server.exe` и `bin/Content.Client/Content.Client.exe` для запуска сервера и клиента соответственно.  

Вот мы вошли - и увидели не лобби, а сразу же станцию на карте dev. Это произошло, потому что билд по умолчанию использует дебаг конфигурацию.  

Чтобы изменить это, используйте: 

* `dotnet build -c Release` - для обычной сборки.  
* `dotnet build -c Tools` - дебаг сборка, но без Assert'ов, которые часто приводят к крашам.

Однако, работая в VS Code, на левой панели у вас присутствует вкладка "Run and Debug".  
В ней присутствуют скрипты, позволяющие автоматически производить описанные выше манипуляции.  

Например:  

* `Client`, `Client (Compatibility renderer)`, `Server` - отдельный запуск клиента и сервера.  
* `Server/Client`, `Release - Server/Client` - дебаг и релиз сборка с последующим запуском сервера и клиента соответственно.

!!! warning
    Важно понимать, что любые изменения в C#-составляющей необходимо сопровождать повторным билдом, дабы те появились в игре.  

    Прототипы же, напротив, этого не требуют - они подгружаются на этапе запуска сервера/клиента, из-за чего изменяются "динамически".
